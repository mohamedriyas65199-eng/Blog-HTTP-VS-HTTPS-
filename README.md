# HTTP vs HTTPS & HTTP Request Methods

A beginner-friendly guide to understanding the difference between HTTP and HTTPS, along with the most commonly used HTTP request methods.

## Overview

This repository covers:
- HTTP vs HTTPS
- Key differences between them
- Common HTTP request methods
- Importance of secure communication

## HTTP vs HTTPS

### HTTP
- Not secure
- No encryption
- Uses Port 80
- Does not require an SSL/TLS certificate

### HTTPS
- Secure
- Encrypts data
- Uses Port 443
- Requires an SSL/TLS certificate

## HTTP Request Methods

- **GET** – Retrieve data from the server.
- **POST** – Send new data to the server.
- **PUT** – Update or replace an existing resource.
- **PATCH** – Partially update an existing resource.
- **DELETE** – Remove a resource from the server.
- **HEAD** – Retrieve only the response headers.
- **OPTIONS** – Show the supported HTTP methods.
- **CONNECT** – Create a secure connection through a proxy.
- **TRACE** – Used for debugging HTTP requests.

## Conclusion

HTTPS is the secure version of HTTP and is recommended for all modern websites. Understanding HTTP request methods helps developers build secure and efficient web applications.
